실행 가능한 구현 범위

실제 API/트래픽 수집은 남겨두고, 앱 동작에 필요한 나머지 요소를 먼저 구현하는 기준 문서

현재 구현 상태

  • 완료 카테고리 슬라이드, 기간 선택, 검색, 랭킹 렌더
  • 완료 상세 바텀시트, 외부 링크 이동 확인 모달
  • 완료 10분 주기 갱신 스케줄러 골격
  • 완료 데이터 소스 분리 구조 (`DataSource.fetchSnapshot`)

미구현 (의도적으로 남김)

  • 보류 실제 플랫폼 API 호출
  • 보류 트래픽 수집/정규화/중복 제거 서버 파이프라인
  • 보류 인증키/쿼터/재시도 정책

API 연결 시 바로 붙일 지점

  • `project/meme-prototype.js`의 `DataSource.fetchSnapshot()`
  • 반환 스키마는 `{ updatedAt, items }` 유지
  • 각 `item`은 `category`, `score`, `detail`, `links` 필드를 포함

다음 구현 순서

  • 1. 서버 API 엔드포인트 먼저 정의
  • 2. 목업 데이터와 동일 스키마로 응답 구현
  • 3. 클라이언트에서 `USE_MOCK_DATA`를 `false`로 전환
