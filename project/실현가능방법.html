2 수집 베이스(현실적인 방향)

필수
공식 API

유튜브/플랫폼 API로 조회수·좋아요·댓글 등 수치 기반 지표 확보.

보조
공개 트렌드

캐시
결과 저장

사용자는 실시간이 아닌 최신 캐시만 조회.

3 10분 갱신 파이프라인

1) 상위 키워드/콘텐츠 목록 선정 (전 회차 상위 + 신규 후보)
2) 공식 API로 수치 업데이트 (조회수/좋아요/댓글)
3) 증가율 계산 (최근 1시간/24시간 비교)
4) 중복 제거 및 키워드 통합 (유사어/해시태그)
5) 랭킹 점수 계산 후 캐시에 저장

4 점수 산식(예시)

  • 기본 점수 = 조회수 로그값 + 좋아요 가중치 + 댓글 가중치
  • 급상승 보정 = 최근 24시간 증가율 × 보정계수
  • 최종 점수 = 기본 점수 + 급상승 보정

5 중복 처리(키워드/콘텐츠)

  • 띄어쓰기/해시태그/영문 표기 통합
  • 유사 키워드 묶기(편집거리/유사도 기준)
  • 동일 콘텐츠 ID 중복 제거

6 운영 현실화 팁

  • 10분마다 전체 수집 대신 “상위 후보만” 증분 업데이트
  • 사용자 화면은 캐시 결과만 노출 (서버 부하 감소)
  • 업데이트 시간 표시로 신뢰 확보
