Meme Radar

최신 밈과 유행을 카테고리별로 훑어보는 앱 설계

로그인 없이 가볍게 들어와 최신 밈을 확인하고, 랭킹을 눌러 요약 정보와 외부 링크로 이동하는 흐름을 핵심으로 설계합니다.

0 프론트엔드 프로토타입

프론트엔드 설계 시안은 별도 파일로 분리했습니다. 아래 파일에서 실제 UI/인터랙션을 확인할 수 있습니다.

프로토타입 파일: project/meme-prototype.html

1 목표

핵심 가치

3초 내 밈 도달

첫 화면에서 3개 카테고리를 슬라이드로 바로 전환합니다.

경험

랭킹에서 바로 이동

순위를 눌렀을 때 요약 정보와 링크를 즉시 제공합니다.

신뢰

출처와 최신성 표기

수집 시간, 플랫폼 출처를 명확히 표시합니다.

2 핵심 사용자

짧게 훑어보는 사용자

  • 이동 중 짧게 확인
  • 즐겨찾기 필수
  • 결정 빠르게 내림

처음 방문한 사용자

  • 튜토리얼 최소화
  • 큰 버튼과 명확한 라벨
  • 실수 복구 쉬움

유행 맥락이 궁금한 사용자

  • 왜 유행인지 요약 필요
  • 가격/인용 정보 확인
  • 원본 링크로 이동

3 정보 구조

홈: 카테고리 슬라이드 3개 (예: 패션 · 영상 · 밈템)
랭킹 리스트: 1~10위 카드형 목록
상세 보기: 요약 → 유행 이유 → 관련 인물/콘텐츠
외부 링크: 구매/원본/출처로 이동

4 주요 기능

카테고리 슬라이드

최대 3개 분야를 좌우 슬라이드로 즉시 전환합니다.

랭킹 카드

순위/요약/핵심 키워드만 노출해 스캔 속도를 높입니다.

외부 링크 연결

카테고리에 따라 가격, 착용자, 원본 링크로 분기합니다.

최신성 표시

업데이트 시간과 출처를 카드에 고정 표기합니다.

5 UI 톤앤매너

시각 언어

  • 큰 제목, 짧은 문장
  • 카드형 랭킹 리스트
  • 슬라이드 전환 강조

모션

슬라이드 전환 0.25초 랭킹 탭 시 하이라이트 상세 진입은 바텀 시트

접근성

  • 텍스트 대비 4.5:1 이상
  • 손가락 터치 영역 44px
  • 링크는 외부 아이콘 표시

6 성능 목표

3초 최초 로딩
60 FPS 스크롤
2시간 랭킹 갱신 주기
20MB 캐시 프리셋

랭킹 기준 설계

수집 소스와 가중치 규칙을 문서화합니다.

외부 링크 정책

구매/원본/출처 링크 구조를 정의합니다.
